# Инструкция для администратора Raspberry Pi

## Что нужно сделать

Добавить **одну строку** в crontab на Raspberry Pi, чтобы данные давлений автоматически отправлялись на сервер каждые 5 минут.

**Зачем:** Сейчас данные забирает промежуточный компьютер (MacBook), который должен быть постоянно включён и подключён к ZeroTier. Когда компьютер выключается — данные на дашборд не поступают. Новая схема убирает эту зависимость: Raspberry Pi сам отправляет данные напрямую на сервер.

**Что НЕ меняется:**
- CoDeSys продолжает работать как раньше
- CSV файлы продолжают генерироваться как раньше
- HTTP-сервер на порту 2224 остаётся (он всё ещё нужен как резерв)
- Никакое ПО не устанавливается

---

## Предварительная проверка

### 1. Проверить что Pi имеет доступ в интернет

```bash
curl -s --max-time 10 https://httpbin.org/ip
```

Ожидаемый результат — JSON с IP-адресом:
```json
{"origin": "203.0.113.42"}
```

Если ошибка или таймаут — Pi не имеет выхода в интернет. Нужно настроить маршрутизацию (через ZeroTier или основной шлюз).

### 2. Проверить что curl установлен

```bash
curl --version
```

Должен показать версию. Если не установлен:
```bash
sudo apt install curl
```

### 3. Найти где лежат CSV файлы

CSV файлы генерируются CoDeSys. Типичные расположения:
```bash
ls -la /var/opt/codesys/PlcLogic/trend/
ls -la /var/opt/codesys/PlcLogic/*.csv
find / -name "*_arc.csv" -maxdepth 5 2>/dev/null | head -10
```

Нам нужен путь к директории с файлами формата `DD.MM.YYYY.{N}_arc.csv`, например:
```
26.03.2026.1_arc.csv
26.03.2026.2_arc.csv
...
```

Запишите этот путь — он нужен для crontab.

### 4. Проверить доступность сервера

```bash
curl -s --max-time 30 -o /dev/null -w "%{http_code}" \
  -X POST https://surgil-dashboard.onrender.com/api/pressure/ingest \
  -H "X-Job-Secret: test"
```

Ожидаемый результат: `401` (сервер отвечает, но секрет неверный — это нормально на этом этапе).

Если таймаут — проблема с сетью. Если `000` — DNS не резолвится.

## Настройка crontab

### Вариант А: Отправка сегодняшних файлов (рекомендуемый)

Открыть crontab:
```bash
crontab -e
```

Добавить в конец файла:

```bash
# Отправка данных давлений на дашборд каждые 5 минут
*/5 * * * * CSV_DIR="/var/opt/codesys/PlcLogic/trend"; DATE=$(date +\%d.\%m.\%Y); for f in ${CSV_DIR}/${DATE}.*_arc.csv; do [ -f "$f" ] && curl -s --max-time 60 -X POST "https://surgil-dashboard.onrender.com/api/pressure/ingest" -H "X-Job-Secret: СЕКРЕТ_БУДЕТ_ПРЕДОСТАВЛЕН" -F "file=@$f" >> /tmp/pressure_ingest.log 2>&1; done
```

**ВАЖНО:** Замените:
- `CSV_DIR="/var/opt/codesys/PlcLogic/trend"` — на реальный путь к CSV файлам
- `СЕКРЕТ_БУДЕТ_ПРЕДОСТАВЛЕН` — на секретный ключ (будет передан отдельно)
- `surgil-dashboard.onrender.com` — на реальный URL приложения

### Вариант Б: Отправка с помощью скрипта (если cron-строка слишком длинная)

Создать файл `/home/vend_1/send_pressure.sh`:
```bash
#!/bin/bash
# Отправка свежих CSV давлений на дашборд
CSV_DIR="/var/opt/codesys/PlcLogic/trend"
API_URL="https://surgil-dashboard.onrender.com/api/pressure/ingest"
SECRET="СЕКРЕТ_БУДЕТ_ПРЕДОСТАВЛЕН"
DATE=$(date +%d.%m.%Y)

for f in ${CSV_DIR}/${DATE}.*_arc.csv; do
    [ -f "$f" ] || continue
    echo "$(date '+%Y-%m-%d %H:%M:%S') Sending $f ..."
    RESULT=$(curl -s --max-time 60 -X POST "$API_URL" \
        -H "X-Job-Secret: $SECRET" \
        -F "file=@$f")
    echo "  Result: $RESULT"
done
```

Сделать исполняемым:
```bash
chmod +x /home/vend_1/send_pressure.sh
```

Добавить в crontab:
```bash
*/5 * * * * /home/vend_1/send_pressure.sh >> /tmp/pressure_ingest.log 2>&1
```

## Обоснование подхода

### Почему curl, а не Python-скрипт?

- `curl` уже установлен на любом Linux (не нужно ставить Python-пакеты)
- Одна строка в crontab — минимальное вмешательство в систему
- Легко отладить: можно запустить команду вручную и сразу увидеть ответ сервера
- Нулевой риск для CoDeSys: curl не трогает файлы, только читает

### Почему HTTPS POST, а не SSH/SCP/rsync?

- Pi за NAT — сервер не может подключиться к Pi
- Pi может подключиться к серверу (исходящий HTTPS)
- HTTPS зашифрован — данные защищены при передаче
- Заголовок `X-Job-Secret` — авторизация (только Pi с секретом может отправлять данные)

### Что если Pi перезагрузится?

- crontab сохраняется при перезагрузке
- После загрузки cron начнёт отправлять файлы через 5 минут
- Данные за время простоя: CSV файлы уже на диске (CoDeSys записал), при первой отправке всё догонится

### Что если интернет пропадёт?

- curl получит таймаут и завершится с ошибкой
- Через 5 минут cron попробует снова
- CSV файлы при этом продолжают записываться CoDeSys
- Когда интернет вернётся — все свежие данные отправятся

### Безопасно ли это для CoDeSys?

**Да.** curl только ЧИТАЕТ CSV файлы. Он не записывает, не удаляет, не блокирует файлы. CoDeSys продолжает работать параллельно без каких-либо ограничений.

### Нагрузка на Pi

- curl запускается на 1-2 секунды каждые 5 минут
- Размер одного CSV: 20-50 KB
- Трафик: ~10 KB/мин (пренебрежимо мало)
- CPU: <1% на время отправки

## Контакты для вопросов

Если что-то не работает — проверить лог:
```bash
cat /tmp/pressure_ingest.log
```

Типичные ошибки:
- `401 Unauthorized` — неверный секрет (X-Job-Secret)
- `Connection timed out` — нет интернета на Pi
- `Could not resolve host` — DNS не работает на Pi
- `400 Bad Request` — файл не соответствует формату (не *_arc.csv)
