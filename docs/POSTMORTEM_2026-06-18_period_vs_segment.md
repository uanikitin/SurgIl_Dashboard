# Постмортем: «круг» Анализ периода ↔ Сегментный анализ (2026-06-18)

> Памятка, чтобы больше не ловить этот цикл «починил одно — сломал другое».

## Симптомы (что наблюдалось ~3 суток)

- В **Сегментном анализе** (страница `/segment-analysis`, шаблон `segment_result.html`)
  колонки давлений `P_шл / P_уст / ΔP / Раб%` показывали `—`.
- В **Анализе периода наблюдения** (визард шаг 3, модал) `Р трубное`/`Р линейное`
  **Среднее** = `—`, пропадала **таблица событий** и **текстовый блок интерпретации**.
- Чинили одно → ломалось другое. За один день 2–3 полных цикла.

## Корневые причины (их было ТРИ разных, отсюда путаница)

1. **Сегмент `—` = застрявший localStorage, НЕ баг расчёта.**
   Страница сохраняет ручные правки сегментов в `localStorage`
   (`segment_edits_{well}_{from}_{to}`). В сохранённых правках НЕТ полей давления
   (`mean_p_*`). При открытии страница перезаписывала серверные сегменты
   (с давлениями) сохранёнными (без давлений) → `—`.
   На **чистой** загрузке (без сохранённых правок) всё считалось правильно.
   *Проверка, которая это вскрыла:* открыл `/segment-analysis` в чистой вкладке —
   давления были на месте; ответ `/api/segment-demo/analyze` содержит
   `chart_data.secondary.{p_tube_mean,p_line_mean,dp_mean}`.

2. **Период `—` / нет событий / нет интерпретации = регрессия от отката.**
   В попытке починить сегмент были откачены 3 файла к старому коммиту `a4d68e1`.
   Но незакоммиченные изменения в этих файлах были не «поломкой», а **свежей
   починкой периода** (функции `_convert_chart_to_chart_data`,
   `_calc_avg_from_chart`, возврат `events_for_chart` / `description` / `describe`,
   таблица «События за период» в шаблоне). Откат удалил эту починку.

3. **Первопричина самого «круга»: общий хрупкий код давлений.**
   Обе фичи опираются на одни и те же данные давлений в разных форматах:
   - API segment-demo: `chart_data.secondary.p_tube_mean.values` (вложенно)
   - customer-daily / снапшот после `_normalizeChartData`: плоские
     `chart_data.p_wellhead / p_flowline / dp`
   Код, читавший только один формат, ломал второй. Правка одной фичи меняла
   общий путь → ломала соседнюю. А проверяли всегда только что-то одно.

## Как починили (итоговое состояние)

- `segment_result.html` — при восстановлении правок из localStorage давления
  **дозаполняются** из исходных (серверных) сегментов по номеру. Ручные правки
  границ/комментарии сохраняются, давления не пропадают.
- `observation_segment_widget.js` — fallback на плоские массивы
  (`chart_data.p_wellhead/...`) рядом с вложенным форматом API.
- `adaptation_report_service.py` — возвращены `_convert_chart_to_chart_data` /
  `_calc_avg_from_chart`, средние давления + `events_for_chart` + `description`.
- `adaptation_wizard.html` — возвращена таблица «📋 События за период».
- Проверено **вживую через браузер сразу на ОБОИХ экранах**:
  период — Скв 98, сегмент — Скв 128. Оба зелёные на одной кодовой базе.

## Правила на будущее (как не повторить)

1. **Проверять ОБА экрана перед фиксацией.** Период и сегмент делят код давлений —
   правка одного требует проверки второго. Минимум: открыть оба и убедиться, что
   давления/события/интерпретация на месте.
2. **Сразу коммитить рабочую точку.** Как только оба зелёные — `git commit`.
   Это якорь, к которому можно вернуться одной командой.
3. **«Верни как было» = git, а не переписывание.** `git log` → `git checkout < commit> -- файл`.
   Не переписывать код заново.
4. **Перед откатом — понять, что теряешь.** Незакоммиченные изменения могут быть
   свежей ПОЧИНКОЙ, а не поломкой. Сначала `git diff` / `diff` с бэкапом.
5. **Странное поведение ОДНОЙ страницы → сначала проверь кэш.** Кнопка
   «🗑 Сбросить редактирование» / очистка `localStorage` ДО правок в коде.
6. **Диагностировать по реальному ответу API (Network), а не по коду.**
   Код говорит «как должно быть», Network — «как есть». Данные давлений были в
   `chart_data.secondary`, а читались не оттуда.
7. **Один формат данных.** Технический долг: договориться об едином формате
   `chart_data` (плоские массивы ИЛИ secondary-объект) и привести все три места
   к нему — тогда общий хрупкий узел исчезнет.
