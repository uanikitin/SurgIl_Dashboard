# Сценарная матрица: плитки методов главы «Адаптация» (шаг 5)

Ручной UX-чек-лист (backend-контракты автоматизированы в
`tests/test_adaptation_methods.py`, 7/7 PASS). Гонять на живом сервере:
`http://localhost:8000/adaptation-report/wizard` → выбрать скважину → Шаг 5.

Легенда: ✅ ожидаемо · ⛔ блокер · ⏳ следующий заход.

---

## Общие правила (любой метод-модалка)

| # | Шаг | Ожидаемо |
|---|-----|----------|
| G1 | Клик по плитке метода | Открывается fullscreen-модалка с заголовком и бейджем метода |
| G2 | Нажать ✕ (справа вверху) | Модалка закрывается, секция возвращается на место (стэш) |
| G3 | Нажать «Закрыть» (футер) | То же, что ✕ |
| G4 | Нажать **ESC** | То же — модалка закрывается, cleanup выполняется ✅ (фикс) |
| G5 | Открыть метод повторно | Содержимое на месте, без дублей/пустот |
| G6 | Открыть метод A, закрыть, открыть метод B | Никаких остатков A в B |

## 🔬 Полный анализ адаптации

| # | Шаг | Ожидаемо |
|---|-----|----------|
| F1 | Открыть плитку | В шапке — **редактируемый период** (📅 from → to), ниже параметры (порог ΔP, окно R★), кнопка ▶ Проанализировать |
| F2 | Изменить период в шапке | Поля Зоны 1 и state синхронизируются (не выходя) |
| F3 | ▶ Проанализировать | Спиннер → результат (карточки, сравнение, графики, события, ИРВ, R★) |
| F4 | Сменить период в шапке и снова ▶ | Пересчёт по новому периоду, без выхода ⛔(было невозможно) |

| F7 | Закрыть (✕/ESC/Закрыть) | Возврат, результат сохранён в state (повторное открытие показывает его) |
| F8 ⏳ | Подразделы результата сворачиваемые; ИРВ/сегментный как отдельные действия | Следующий заход |

## Известные ограничения (честно)

- **Галочки артефактов в Зоне 3 не персистятся в БД/PDF**: customer-daily
  `PUT /blocks/{id}` намеренно игнорирует `params` (защита снапшота,
  `customer_daily.py:499`). Сейчас галочки меняют только client-render превью.
  Полная персистентность → задача APIEngineer (whitelist `params.parts`).
  Зафиксировано тестом `test_blocks_crud_and_params_protection`.
- F8 (подразделы + отдельные ИРВ/сегментный) — следующий заход.
- e2e в браузере (Playwright) не настроен; UX-проверка — по этой матрице вручную.
