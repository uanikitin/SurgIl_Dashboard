# Старт нового чата Claude Code — со структурным анализом и графом

Как начать новый чат, чтобы он использовал **структурный анализ ([[CODEMAP]])**
и **граф проекта (`graphify-out/`)**.

## Два механизма (разное назначение)

| | Что | Как чат использует |
|---|---|---|
| **CODEMAP.md** | структурный анализ: что править вместе, инварианты, что не трогать | читается **автоматически** (пойнтер в `CLAUDE.md`), лучше попросить явно |
| **Граф `graphify-out/`** | связи между 287 файлами (узел = файл) | по запросу: `graphify query "…"` или `graph.html` |

## Первое сообщение в новый чат (копировать)

```
Перед работой прочитай по порядку: CLAUDE.md → CODEMAP.md → HANDOFF.md.

Для связей между файлами используй граф проекта (graphify-out, модульный, 287 файлов):
команда `graphify query "<файл или фича>"` возвращает все связанные файлы.
Например перед правкой отчёта: graphify query "adaptation report service".
Визуально граф — graphify-out/graph.html. GRAPH_REPORT.md устарел, не используй.

Задача: <твоя задача>. Стартовые файлы и связность — в HANDOFF.md и CODEMAP.
```

## Как это работает на практике

- **CODEMAP** даёт правила и «правится вместе» сразу, без запросов — первичный
  инструмент для кода. Разделы: §0a маски/слои давления, §3 отчёты, §4 observation.
- **Граф** отвечает на «что зацепит правка файла X»:
  - `graphify query "adaptation report service"` → ~86 связанных файлов
    (daily_report, full_pipeline, customer_daily, observation, wizard…).
  - `graphify path "fileA" "fileB"` — кратчайший путь между файлами.
  - `graphify explain "<файл>"` — пояснение узла.
- **graph.html** — открыть в браузере, видеть подсистемы глазами.

## Оговорки

- Граф **модульный** (узел = файл, связь = зависимость). Для архитектуры/связности —
  идеально; функции/классы не показывает.
- ⚠️ `graphify-out/GRAPH_REPORT.md` — **СТАРЫЙ** (символьный граф до пересборки),
  не использовать. Актуальны `graph.json` / `graph.html` / `graphify query`.
- Пересобрать модульный граф: `python graphify-out/build_module_graph.py`.
- Память (`MEMORY.md`) загружается сама — там пойнтеры на [[HANDOFF]], [[CODEMAP]]
  и доменку масок.

См. также: [[CODEMAP]], [[HANDOFF]].
